RAG Chatbot — Frequently Asked Questions (FAQ)
================================================

Q: What is a RAG chatbot?
A: RAG stands for Retrieval-Augmented Generation. It is a technique where an AI model first retrieves relevant passages from a document (or database) and then generates an answer based only on those retrieved passages. This prevents the AI from hallucinating facts that are not in the document.

Q: What file formats does this chatbot support?
A: The chatbot supports PDF and TXT files. You can upload any plain-text TXT file or a standard PDF document. The maximum file size is 10 MB.

Q: How does the document chunking work?
A: When you upload a document, it is split into smaller overlapping text chunks. Each chunk is approximately 500 words with a 50-word overlap with the adjacent chunk. This overlap ensures that sentences at chunk boundaries are not lost.

Q: What embedding model is used?
A: The chatbot uses the sentence-transformers model "all-MiniLM-L6-v2" to convert text chunks into vector embeddings. These vectors are stored in a ChromaDB vector database for fast similarity search. This model runs locally — no external API call needed for embeddings.

Q: What language model generates the answers?
A: Answers are generated by Llama 3.1 8B (via the Groq API), which is extremely fast. It receives the top-k most relevant chunks as context and is instructed to answer only from that context.

Q: What does "source citation" mean?
A: After each answer, the chatbot shows which chunks (by filename and chunk number) were used to generate the answer, along with a similarity score expressed as a percentage. A higher percentage means the chunk was more relevant to your question.

Q: Can I upload multiple documents?
A: Yes. You can upload multiple documents in the same session. Each document gets its own ID. You can click on a document in the sidebar to make it the active context for questions, or ask across all documents by not selecting a specific one.

Q: How do I clear all documents?
A: Click the "Clear all documents" button at the bottom of the sidebar. This will delete all stored embeddings from ChromaDB and reset the chat.

Q: Is my data sent to any external server?
A: Your questions and retrieved chunks are sent to the Groq API for answer generation. Embeddings are computed locally using sentence-transformers — your document text never leaves the server for that step. No data is sent anywhere else.

Q: What happens if the answer is not in the document?
A: If the relevant information is not found in the uploaded document, the chatbot will respond: "I couldn't find an answer to that in the uploaded document." It will not make up information.

Q: Can I use this chatbot without a Groq API key?
A: No. The chatbot requires a Groq API key to generate answers. Embeddings are handled locally so no OpenAI key is needed. You must provide your GROQ_API_KEY in the .env file before starting the server. Get a free key at console.groq.com.

Q: How do I run this chatbot locally?
A: Install dependencies with "pip install -r requirements.txt", create a .env file with your GROQ_API_KEY, then run "uvicorn app.main:app --reload". The app will be available at http://localhost:8000.

Q: What is the top-k parameter?
A: TOP_K controls how many text chunks are retrieved from ChromaDB for each question. The default is 4. Higher values give the model more context but may include less relevant information.

Q: Does the chatbot support streaming responses?
A: Yes. Responses are streamed using Server-Sent Events (SSE), so you see the answer appear word-by-word in real time, similar to ChatGPT.

Q: What is ChromaDB?
A: ChromaDB is a lightweight, open-source vector database that runs locally without any external service. It stores the text chunks and their embeddings, and supports fast nearest-neighbor search to find the most relevant chunks for a given question.

Q: How is similarity measured?
A: ChromaDB uses cosine distance to measure how similar a query embedding is to stored chunk embeddings. A score of 100% means perfect similarity; lower scores indicate less relevance.

Q: Can I deploy this to a cloud service?
A: Yes. The project includes a render.yaml for deployment on Render.com. The server starts with "uvicorn app.main:app --host 0.0.0.0 --port $PORT". Make sure to set the OPENAI_API_KEY environment variable in your cloud provider's dashboard.

Q: What should I do if the upload fails?
A: Common causes are: file format not supported (only PDF/TXT), file size over 10 MB, or an invalid/expired OpenAI API key. Check the toast notification for the specific error message.

Q: Is the chat history saved?
A: Chat history is stored only in the browser session (in memory). If you refresh the page, the chat history will be lost. However, uploaded documents remain in ChromaDB until you click "Clear all documents" or restart the server.
